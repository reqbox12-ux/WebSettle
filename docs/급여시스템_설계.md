**insurance_payments 테이블 구조:**
```sql
CREATE TABLE insurance_payments (
    id           INTEGER PRIMARY KEY,
    year         INTEGER,
    month        INTEGER,
    branch       TEXT,
    pension_co   INTEGER,   -- 국민연금 본사분
    pension_emp  INTEGER,   -- 국민연금 직원분
    health_total INTEGER,   -- 건강보험 합산 (50/50 자동분할)
    employ_co    INTEGER,   -- 고용보험 본사분
    employ_emp   INTEGER,   -- 고용보험 직원분
    accident     INTEGER,   -- 산재보험 (전액 본사)
    UNIQUE(year, month, branch)
)
```

**보험료 분할 로직:**
- 건강보험: `health_total ÷ 2` = 본사분, 나머지 = 직원분
- 국민연금 / 고용보험: 고지서에서 직접 분리 입력
- 산재보험: 전액 본사 부담

## 3. 현재 급여 업로드 구조 (변경 예정)

### 현재 방식 (deprecated 예정)

`데이터 업로드 > 인건비 > 지점별 대시보드.xlsx` 업로드

| 시트명 | 내용 | 파서 |
|--------|------|------|
| `지점별집계` | 4대보험 가입자 급여 (지점별 집계) | `parse_payroll_insured()` |
| `사업소득자` | 프리랜서 급여 (지점별 집계) | `parse_payroll_freelance()` |

**저장 테이블: `payroll`**
```sql
payroll: branch, type(insured/freelance), gross_pay, net_pay,
         insurance(직원부담), income_tax, local_tax, headcount
```

### 문제점
- 지점별 집계 파일 → 직원별 상세 내역 없음
- 본사 부담 4대보험 미포함
- 급여명세서 생성 불가
- 소득세/지방세 합산으로만 관리

## 4. 변경 후 계정 구조

### 새 인건비 계정 항목

| 계정 | 내용 | 데이터 출처 |
|------|------|-----------|
| **직원 실지급액** | 정규직 실수령액 합계 | 급여 계산 결과 |
| **4대보험료_직원** | 직원 부담 4대보험 합계 | 4대보험 고지서 입력 |
| **4대보험료_본사** | 본사 부담 4대보험 합계 | 4대보험 고지서 입력 |
| **사업소득자 실지급액** | 프리랜서 실수령액 합계 | 급여 계산 결과 |
| **소득세** | 사업소득자 소득세 (3%) | 자동 계산 |
| **지방세** | 사업소득자 지방세 (0.3%) | 자동 계산 |

### 인건비합계 계산식

```
인건비합계 = 직원실지급액
           + 4대보험료_직원
           + 4대보험료_본사
           + 사업소득자실지급액
           + 소득세
           + 지방세
```

## 5. 급여 시스템 전체 설계

### 전체 흐름

```
[1단계: 직원 마스터 등록] ← 최초 한 번, 이후 신규 추가만
  직원명 + 소속지점 + 유형(정규직/사업소득자) + 부양가족수

          ↓ 매월

[2단계: 급여 계산]
  세전금액 입력 (직원별)
       ↓
  자동 계산
  ┌────────────────────────────────────┐
  │ 정규직                              │
  │  소득세 → 간이세액표 조회 (부양가족수) │
  │  지방세 = 소득세 × 10%              │
  │  실지급액 = 세전 - 소득세 - 지방세   │
  │  (4대보험은 별도 고지서 입력)        │
  ├────────────────────────────────────┤
  │ 사업소득자                          │
  │  소득세 = 세전 × 3%                │
  │  지방세 = 세전 × 0.3%              │
  │  실지급액 = 세전 - 소득세 - 지방세   │
  └────────────────────────────────────┘
       ↓
  급여 계산 결과 → payroll_entries 테이블 저장

[3단계: 4대보험 입력]
  월별 고지서 기준 입력
  (국민연금/건강보험/고용보험/산재 → 본사분/직원분 분리)
  → insurance_payments 테이블 저장

[4단계: 급여명세서 출력]
  직원별 PDF 생성
  이메일 발송 (직원 이메일 등록 시)

[5단계: 지점별 집계 연동]
  계산 결과 → 기존 payroll 테이블에 집계값 저장
  → 대시보드 손익 자동 반영
```

## 6. 새로 필요한 테이블

### `employees` — 직원 마스터

```sql
CREATE TABLE employees (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    name         TEXT NOT NULL,           -- 직원 이름
    branch       TEXT NOT NULL,           -- 소속 지점
    type         TEXT NOT NULL,           -- 'insured'(정규직) / 'freelance'(사업소득자)
    dependents   INTEGER DEFAULT 1,       -- 부양가족 수 (소득세 계산용)
    email        TEXT DEFAULT '',         -- 급여명세서 발송용
    note         TEXT DEFAULT '',         -- 비고
    created_at   TEXT DEFAULT (datetime('now','localtime'))
    -- 퇴직자도 삭제하지 않고 유지 (급여 내역 매칭 보존)
)
```

### `payroll_entries` — 월별 직원별 급여 내역

```sql
CREATE TABLE payroll_entries (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    year         INTEGER,
    month        INTEGER,
    employee_id  INTEGER,                 -- employees.id 참조
    branch       TEXT,                   -- 집계 편의를 위해 비정규화
    type         TEXT,                   -- 'insured' / 'freelance'
    gross_pay    INTEGER DEFAULT 0,      -- 세전 금액
    income_tax   INTEGER DEFAULT 0,      -- 소득세
    local_tax    INTEGER DEFAULT 0,      -- 지방세
    net_pay      INTEGER DEFAULT 0,      -- 실지급액
    note         TEXT DEFAULT ''
)
```

### `tax_brackets` — 간이세액표

```sql
CREATE TABLE tax_brackets (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    year          INTEGER,               -- 적용 연도
    income_from   INTEGER,               -- 월 급여 구간 시작
    income_to     INTEGER,               -- 월 급여 구간 끝
    dependents_1  INTEGER,               -- 부양가족 1명일 때 세액
    dependents_2  INTEGER,               -- 부양가족 2명일 때 세액
    dependents_3  INTEGER,               -- 부양가족 3명일 때 세액
    dependents_4  INTEGER,               -- 부양가족 4명일 때 세액
    dependents_5  INTEGER                -- 부양가족 5명 이상일 때 세액
)
```

> 간이세액표 데이터는 국세청 고시 기준, 사용자가 엑셀/파일로 제공 예정

## 9. 사업소득자 세금 계산

```python
def calc_freelance(gross: int) -> dict:
    income_tax = round(gross * 0.03)      # 소득세 3%
    local_tax  = round(gross * 0.003)     # 지방세 0.3%
    net_pay    = gross - income_tax - local_tax
    return {
        "gross_pay":   gross,
        "income_tax":  income_tax,
        "local_tax":   local_tax,
        "net_pay":     net_pay,
    }
```

## 10. 정규직 소득세 계산 (간이세액표)

```python
def calc_insured(gross: int, dependents: int, year: int) -> dict:
    # tax_brackets 테이블에서 해당 구간 조회
    income_tax = get_tax_from_bracket(gross, dependents, year)
    local_tax  = round(income_tax * 0.1)
    net_pay    = gross - income_tax - local_tax
    # 4대보험 공제는 별도 고지서 입력이므로 여기서 제외
    return {
        "gross_pay":   gross,
        "income_tax":  income_tax,
        "local_tax":   local_tax,
        "net_pay":     net_pay,
    }
```

> ⚠️ 4대보험 직원분은 급여명세서에는 표시되지만, 실제 금액은 매월 고지서에서 따로 입력

### ② 간이세액표 데이터
- 국세청 간이세액표 파일 (엑셀/PDF) 있는지?
- 있으면 가져오면 JSON 변환 작업 진행

### ③ 기존 payroll 데이터 처리
- 현재 DB에 저장된 payroll 데이터 어떻게 할지?
- **A안:** 기존 데이터 유지 + 신규부터 새 방식 적용
- **B안:** 기존 데이터 삭제 + 새 파일로 재입력
- 어느 쪽이 나은지 결정 필요

### ④ 4대보험 입력 위치
- 현재: `데이터 업로드 > 인건비 > 4대보험 부담금` 탭에 있음
- 향후: `급여 관리 > 4대보험 입력` 탭으로 이동 예정
- 이동 시 기존 업로드 탭에서 제거

### ⑤ 급여 지급일
