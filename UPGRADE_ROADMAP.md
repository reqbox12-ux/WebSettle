# WebSettle 업그레이드 로드맵

라온스포츠 통합 운영 플랫폼으로의 확장 계획을 단계별로 정리한 문서입니다.

---

## 현재 시스템 (WebSettle v1)

**핵심 기능**
- 22개 지점 손익 데이터 통합 (카드·통장·급여)
- 직원 마스터 관리 (4대보험·사업소득자·사업자)
- 월별 급여 계산 및 급여명세서 발행
- 4대보험 공단 고지내역 연동
- 지점별 주소·위치(위도/경도) 및 카카오맵 연동
- 자동 거래 분류 규칙 관리

**기술 스택**
- Frontend: Streamlit (Python)
- Database: SQLite (`data/settlement.db`)
- 데이터 처리: Pandas + Openpyxl
- 배포: NAS Docker 컨테이너

## Phase 1 — 지점 운영 기반
> 출퇴근 관리 + 인사 자동화 | 현재 시스템 위에 바로 확장 가능

### 1-1. 지점 전용 랜딩페이지
- 지점별 독립 URL (`/branch?id=수원점` 또는 서브도메인)
- 관리자 앱과 완전 분리된 직원용 경량 UI
- 모바일 최적화 (터치 친화적 출퇴근 버튼)

### 1-2. 직원 인증
- 4~6자리 PIN 또는 사번 로그인 (관리자 계정과 별도 테이블)
- QR코드 방식 인증 (매달 갱신되는 지점별 QR)
- 로그인 유지 시간 설정 (출퇴근 전용 기기는 상시 유지)

### 1-3. GPS 출퇴근
- 지점 좌표(현재 DB에 이미 저장됨) 반경 내에서만 출퇴근 허용
- 위치 이탈 시 경고 표시, 관리자 알림
- 출근/퇴근 버튼 → 타임스탬프 저장, 같은 날 중복 출근 방지

### 1-4. 근태 관리
- 월간 출결 캘린더 (출근/지각/조퇴/결근/휴가 색상 구분)
- 지각·조퇴 자동 플래그 (근무 시작·종료 시간 기준)
- 휴게시간 자동 차감 (8시간 이상 시 30분, 근로기준법 준수)
- 이상 기록 감지 (퇴근 누락, 12시간 초과 근무 등)

### 1-5. 급여 자동 연동
- 직원 마스터에 `시급(hourly_rate)` 컬럼 추가
- 월 누적 근로시간 × 시급 = 세전급여 자동 산출
- 초과근무 1.5배 자동 적용
- 최저임금 미달 시 경고
- 산출 결과가 현재 급여 계산 시스템에 자동 연동

### 1-6. 직원 셀프서비스
- 자신의 급여명세서 직접 조회
- 연차 잔여 일수 확인
- 근태 기록 조회 (본인 분)

### 1-7. 연차 관리
- 입사일 기준 연차 자동 부여 (1년 미만 월 1일, 1년 이상 15일~)
- 연차 사용 신청 → 관리자 승인
- 잔여 연차 실시간 조회

### 관리자 추가 뷰
- 실시간 현황판: 지금 각 지점 출근 인원
- 지점별 인건비 예측: 누적 근로시간 기반 당월 인건비 추정
- 전체 근태 리포트 엑셀 다운로드

## Phase 2 — 본사-지점 연결
> 운영 요청 + 재고 관리 | Phase 1과 병행 개발 가능

### 2-1. AS 요청 시스템
- 지점에서 시설 고장·수리 요청 등록 (제목, 내용, 사진 첨부)
- 본사 담당자 배정
- 처리 상태 추적: `접수 → 배정 → 처리중 → 완료`
- 처리 완료 시 지점 담당자에게 알림

### 2-2. 비품 구매 요청
- 지점에서 필요 물품 요청 등록 (품목, 수량, 사유)
- 본사 승인 / 반려 (반려 사유 입력)
- 구매 처리 후 납품 일자 등록
- 요청~완료 전체 이력 관리

### 2-3. 재고 관리
- 지점별 소모품·비품 재고 등록 및 수량 추적
- 출고 기록 (누가, 언제, 얼마나)
- 최소 재고 수량 설정 → 부족 시 자동 알림 및 구매 요청 연동
- 월별 소모량 리포트

### 2-4. 공지·공문 관리
- 본사 → 전체 또는 특정 지점 공지 발송
- 공지 확인 여부 추적 (읽음/안 읽음)
- 공문 첨부파일(PDF) 관리

## Phase 3 — 회원 관리 CRM
> 지점의 핵심 비즈니스 | Phase 2 이후 착수 권장

### 3-1. 회원 등록 및 관리
- 기본 정보: 이름·연락처·생년월일·성별·가입일·담당지점
- 회원 상태: 활성·휴면·탈퇴
- 사진 등록 (신분 확인용)
- 메모 및 특이사항 기록

### 3-2. 회원권 등록
- 종류: 헬스·PT·GX·골프레슨·복합 등 지점별 상품 연동
- 유효기간 또는 횟수 기반 관리
- 복수 회원권 동시 보유 가능
- 일시 정지(홀드) 기능, 정지 기간만큼 유효기간 자동 연장

### 3-3. 수업 예약 및 등록
- 회원 ↔ 강사·트레이너 매칭
- 수업 시간표 등록 (요일·시간·정원)
- 예약·취소·대기 관리
- 수업 완료 시 잔여 횟수 자동 차감

### 3-4. 강사·트레이너 수업 관리
- 강사별 담당 회원 목록
- 월별 수업 횟수 집계 → 급여 연동 가능 (수업당 단가 설정)
- 수업 노트 기록 (회원별 운동 기록, 레슨 피드백)

### 3-5. 회원 통계
- 지점별 활성 회원 수
- 회원권 종류별 가입 현황
- 신규/탈퇴 트렌드 (월별)
- 방문 빈도 분석, 이용률 리포트

### 3-6. 만료 알림 자동화
- 회원권 만료 D-7, D-1 자동 문자 발송
- 장기 미방문 회원 자동 감지 (30일/60일 미방문)
- 재등록 유도 메시지 자동 발송

## Phase 4 — 판매 및 결제 시스템
> 직접 수익 채널 | Phase 3 완료 후 착수 권장

### 4-1. PG사 연동
- 연동 후보: 토스페이먼츠, KG이니시스, 나이스페이
- 회원권·상품 온라인 결제
- 카드·계좌이체·간편결제(카카오페이·네이버페이) 지원
- 결제 결과 WebHook → 회원권 자동 발급

### 4-2. 지점 상품 등록
- 지점별 판매 물품 등록 (단백질 쉐이크, 운동 용품, 음료 등)
- 상품별 가격·재고 관리
- 재고 Phase 2 재고 시스템과 연동

### 4-3. POS 기능
- 현장 카드·현금 결제 기록
- 일별 매출 집계
- 영수증 출력 또는 문자 발송

### 4-4. 매출 자동 연동
- 결제 내역이 현재 손익 시스템(branch_monthly_revenue)에 자동 반영
- 카드·현금·온라인 결제 채널별 매출 분리 집계

### 4-5. 환불·취소 처리
- 결제 취소 이력 관리
- 부분 환불 지원
- 환불 시 회원권 잔여 횟수·기간 복원

## Phase 5 — 문자 및 마케팅
> 회원 커뮤니케이션 | Phase 3+4 이후 최대 효과

### 5-1. 문자 발송 시스템
- 연동 후보: 카카오 알림톡, 솔라피(SOLARAPI), NHN Cloud SMS
- 개별 발송 및 대량 발송
- 발송 이력 로그 (일시·수신자·내용·결과)

### 5-2. 대상 필터링
- 지점별 / 회원권 종류별 / 만료 임박자 / 장기 미방문자
- 성별·연령대 필터
- 복합 조건 AND/OR 설정

### 5-3. 템플릿 관리
- 자주 쓰는 문자 템플릿 저장·편집
- 변수 치환 지원 (`{이름}`, `{만료일}`, `{잔여횟수}` 등)
- 카카오 알림톡 템플릿 승인 관리

### 5-4. 자동 발송 설정
- 생일 당일 축하 메시지 자동 발송
- 회원권 만료 알림 자동 발송 (D-7, D-3, D-1)
- 장기 미방문 리마인드 자동 발송
- 발송 시간 예약

### 5-5. 마케팅 효과 분석
- 문자 발송 후 재등록률 추적
- 캠페인별 전환율 분석
- 발송 비용 대비 수익 리포트

## 기술 아키텍처 업그레이드 방향

### 현재 → Phase 1~2
```
Streamlit + SQLite 그대로 유지 (확장만)
지점 랜딩페이지: Streamlit multi-page 또는 별도 경량 Flask앱
```

### Phase 3 이후 (데이터 볼륨 증가 시)
```
Database: SQLite → PostgreSQL (Neon / Railway / 자체 서버)
Backend: Streamlit API 로직 → FastAPI 분리
Frontend: 직원·회원용은 React 또는 Next.js 경량 앱 검토
인증: 현재 세션 방식 → JWT 토큰 방식으로 전환
```

### Phase 4 이후 (결제·외부 연동)
```
PG사 WebHook 수신: 독립 FastAPI 엔드포인트
문자 발송: 비동기 처리 (Celery + Redis 또는 Cloud Task Queue)
파일 저장: 로컬 → S3 호환 오브젝트 스토리지 (사진·첨부파일)
```

## 권장 개발 순서

```
현재 (v1)
│
├─ Phase 1: 출퇴근·인사 자동화          ← 즉시 착수 가능
│
├─ Phase 2: 본사-지점 운영 연결          ← Phase 1과 병행 가능
│
└─ Phase 3: 회원 CRM                    ← Phase 1·2 안정화 후
    │
    ├─ Phase 4: 결제 시스템              ← Phase 3 이후
    │
    └─ Phase 5: 문자·마케팅             ← Phase 3+4 이후 최대 효과
```

## 우선순위 요약

| 순위 | Phase | 이유 |
|------|-------|------|
| 1 | Phase 1 (출퇴근) | 현재 급여 시스템과 직결, 즉시 ROI |
| 2 | Phase 2 (운영 요청) | 본사-지점 소통 비효율 즉시 해소 |
| 3 | Phase 3 (CRM) | 회원 데이터가 모든 후속 기능의 기반 |
| 4 | Phase 4 (결제) | Phase 3 없이는 의미 없음 |
